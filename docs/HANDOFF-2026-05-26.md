## 🛡 Patterns sécurité production-grade appliqués

- **Validation stricte** RFC standards (RFC 1035 hostnames, RFC 5322 emails côté forms publics)
- **Error codes stables** exportés par module (constants frozen, mapping UI possible sans coupling)
- **Edge cases** : null/NaN/empty/overflow/division 0 systématiquement testés
- **Mocks FLAG INACTIF** réalistes : shape cohérent quand creds absents (dns_records[], dns.google response, OpenAI completion stub, Stripe Connect mock)
- **PII scrub Loi 25** : emails/phones/CC/IP regex strip avant envoi tiers (OpenAI surtout)
- **IP hashing SHA-256** : anti-spam anonymes (surveys, reviews, webchat) — IP jamais en clair
- **Rate-limit KV sliding window** : 30/min chat, 60/min anti-bot, fail-open si KV down
- **Anti-XSS** : sanitizeBody strip `<script>` `<iframe>` `on*=` `javascript:`
- **A11y WCAG 2.1 AA** : aria-busy/aria-live/role=alert/aria-label sur toutes UI
- **100% additif** : zéro régression sur 186 fichiers tests, signatures handlers préservées
